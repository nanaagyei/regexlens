# Getting Started

**RegexLens** is an open-source, UI-first developer tool for **JavaScript regular expressions** (`RegExp`). It helps you see **what** matches, **why** it matches (plain-English steps), **how** the pattern is structured (AST + railroad diagram), and **where** it may be risky (smart warnings). Optional sign-in adds cloud-backed features like a saved library, exports, deeper **Analysis**, and **Regex Copilot** (streaming AI), depending on how you deploy and configure the app.

## Where things live

| Surface | URL / path | Purpose |
|--------|------------|---------|
| Marketing site | `/` | Landing and sign-in entry |
| Workbench | `/app` | Full editor: pattern, test text, matches, tabs |
| Docs (this site) | [docs.regexlens.dev](https://docs.regexlens.dev) — Nextra in `docs/`; local dev port `3002` | Long-form help |
| Changelog (web) | [regexlens.dev/changelog](https://regexlens.dev/changelog) | High-level release notes |

Set `NEXT_PUBLIC_SITE_URL` and `NEXT_PUBLIC_DOCS_URL` in production so share links, metadata, and documentation links use your domains (defaults: `https://regexlens.dev` and `https://docs.regexlens.dev`).

## Anonymous vs signed-in (at a glance)

| Capability | Anonymous | Signed in |
|------------|:---------:|:---------:|
| Live matching, flags, templates, fixtures | Yes | Yes |
| Explanations, AST, railroad, warnings | Yes | Yes |
| Shareable URLs | Yes | Yes |
| Saved library, versions, diff | — | Yes |
| Export (Markdown, PR comment, Notion, plain) | — | Yes |
| **Analysis** tab + `POST /api/analyze` | — | Yes * |
| **Regex Copilot** (`/api/ai/chat`) | — | Yes * |

\* Requires your deployment to configure the relevant server credentials (for example `ANTHROPIC_API_KEY`).

Signing in is optional for core editing. OAuth (GitHub, Google) and magic links (Resend) are available via Auth.js on the main app when enabled.

## Workbench layout

1. **Pattern** (left) — Edit regex, toggle flags (`g`, `i`, `m`, `s`, `u`, `y`), token toolbar, parse status.
2. **Test text** + **Matches** (center) — Sample input, highlighted matches, capture groups; fixture suite panel when you load a suite.
3. **Tabs** (right) on desktop; **segmented control** on small screens:
   - **Explain** — Deterministic step-by-step explanation from the AST (not LLM output).
   - **Structure** — Collapsible AST tree synced with hover elsewhere when applicable.
   - **Railroad** — Syntax railroad diagram for the pattern.
   - **Analysis** — Risk score, complexity, extra warnings/suggestions via server analysis (signed in).
   - **Warnings** — In-app static warnings (severity info / warn / danger).
   - **Copilot** — Streaming assistant with actions like apply pattern (signed in; provider dependent).

Header: **Docs** (opens documentation), **Templates**, **Fixtures**, **Save** / **Library**, **Share**, **Reset**, account menu, and **Homepage** returns to marketing `/`.

## Sharing & URL state

The editor syncs pattern, flags, and test text into query parameters (`p`, `f`, `t`). Values are typically **base64**-encoded (with URI-safe handling) so links stay portable. Anyone with the link can load the same state—**do not put secrets** in pattern or test text if you share URLs.

See [Sharing & URLs](/features/sharing).
